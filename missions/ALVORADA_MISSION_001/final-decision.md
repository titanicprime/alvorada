# ALVORADA MISSION 001 — FINAL DECISION

Build a small problem-solving asset store.

Each completed task may contribute:
- one useful method;
- one useful failure pattern.

Each new task may retrieve:
- up to two relevant methods;
- any directly relevant failure pattern.

For every retrieved item, record:
- retrieved;
- used;
- helped, harmed, neutral, or unclear.

First implementation:
- begin with failure patterns only;
- check the new task against prior failure patterns at FRAME;
- surface relevant warnings;
- record whether each warning was used;
- at task completion, record whether the failure recurred;
- add at most one new failure pattern.

Pilot:
- run for five real Alvorada tasks;
- then decide KEEP, REVERT, or EXTEND.

Authority:
- workers may propose assets;
- Red-1 may collect and present them;
- André decides what remains active guidance;
- no automatic canonization;
- no automatic standing changes.

Status: ACCEPTED
